# PCBA 工艺路线域（Process Route Bounded Context）

> **限界上下文**：工艺路线域  
> **统一语言**：ProcessRoute, RouteStep, RouteVersion, RouteTemplate, RouteGraph, StepInstance  
> **核心关注**：定义工序如何组合为工艺路线，包括路线定义模型、版本管理、5种标准路线模板及双面混装路线的27个步骤实例详述  
---

## 领域概述

工艺路线域是 PCBA MES 系统的**核心域**，实现"模型优先"的设计目标——通过路线组合框架将 **工序域** 定义的27个标准工序类型组合为多种工艺路线模板，一套模型支撑多种产品工艺路线。

本域分为三个层次：
1. **路线组合框架**（原第7章）：定义路线的数据模型、步骤实例结构、路线图语义和版本管理
2. **标准路线模板**（原第8章）：定义5种标准模板及对比矩阵，支撑不同产品类型的工艺路线需求
3. **双面混装详述**（原第9章）：对最复杂的双面混装路线进行27个步骤实例的逐一详述

**跨领域协作**：
- 本域的路线步骤引用 **工序域** 的工序类型定义
- 本域的路线步骤中挂载 **质量域** 的质量门禁
- 本域的路线绑定到 **制造实体域** 的产线和工单
- 本域的步骤实例中定义 **数据采集域** 的数据采集点
- 本域的路线步骤中引用 **物料域** 的物料消耗

# 路线组合框架

路线（Route）是工艺管理的核心编排对象，它将工序类型按生产逻辑串联为一条可执行的加工路径。本章定义路线的数据模型、步骤实例化规则、路线图语义及版本管理机制。

---

## 7.1 路线定义模型

路线定义（RouteDefinition）描述一条工艺路线的元信息，包括编码标识、适用产品范围、所包含的步骤图结构以及生命周期状态。其数据模型如下：

| 属性 | 数据类型 | 必填 | 说明 |
|------|---------|------|------|
| routeCode | String(64) | 是 | 路线编码，全局唯一，格式 `ROUTE.{产品族}.{工艺特征}.{序号}`，如 `ROUTE.PCBA.DBL_MIXED.001` |
| routeName | String(128) | 是 | 路线中文名称，如"双面混装SMT+THT波峰焊标准路线" |
| version | String(16) | 是 | 语义化版本号，格式 `MAJOR.MINOR.PATCH`，如 `2.1.0` |
| status | Enum | 是 | 路线状态：`DRAFT`（草稿）、`REVIEWED`（已评审）、`RELEASED`（已发布）、`OBSOLETE`（已废弃） |
| productFamily | String(64) | 是 | 适用产品族编码，如 `PCBA-CTRL-BOARD` |
| productModels | String[] | 否 | 适用具体产品型号列表；为空表示适用于整个产品族 |
| stepGraph | StepGraph | 是 | 步骤图对象，包含步骤实例集合及连接关系 |
| effectiveDate | DateTime | 否 | 路线生效时间，为空表示审批通过即时生效 |
| expirationDate | DateTime | 否 | 路线失效时间，为空表示长期有效 |
| createdBy | String(64) | 是 | 创建人工号 |
| approvedBy | String(64) | 否 | 审批人工号，`RELEASED` 状态时必填 |
| createdAt | DateTime | 是 | 创建时间 |
| updatedAt | DateTime | 是 | 最后修改时间 |
| remark | String(512) | 否 | 备注说明 |

**状态流转规则：**

```
DRAFT ──(提交评审)──> REVIEWED ──(审批通过)──> RELEASED ──(新版本替代)──> OBSOLETE
  │                       │                        │
  └──(编辑)──> DRAFT      └──(驳回)──> DRAFT       └──(紧急召回)──> OBSOLETE
```

- 只有 `RELEASED` 状态的路线可被生产工单引用
- `OBSOLETE` 状态的路线仅保留历史追溯，不可新建工单

## 7.2 步骤实例定义

步骤实例（StepInstance）是工序类型在具体路线中的实例化产物。同一工序类型（如 `STEP_TYPE.SMT.AOI`）可在同一路线中被多次实例化（如 T面 AOI 和 B面 AOI），每次实例化携带不同的参数配置和上下文。

| 属性 | 数据类型 | 必填 | 说明 |
|------|---------|------|------|
| instanceId | String(16) | 是 | 步骤实例ID，路线内唯一，格式 `S{N}`，如 `S1`、`S12` |
| stepType | String(64) | 是 | 引用的工序类型编码，来自工序元素库，如 `STEP_TYPE.SMT.PRINT` |
| instanceName | String(64) | 是 | 步骤实例名称，区分同一工序类型的不同实例，如"T面锡膏印刷"、"B面锡膏印刷" |
| executionSide | Enum | 是 | 执行面：`TOP`（T面/元件面）、`BOTTOM`（B面/焊面）、`NA`（不适用） |
| sequenceNo | Integer | 是 | 序列号，路线上展示排序用 |
| predecessors | String[] | 是 | 前驱步骤实例ID列表；线性步骤为单个前驱，合并步骤可有多个前驱 |
| successors | String[] | 是 | 后继步骤实例ID列表；分支步骤可有多个后继 |
| boundQualityGate | String(32) | 否 | 绑定的质量门禁编码，如 `QG-SMT-AOI-01`；门禁未通过时阻断流转 |
| parameterOverrides | Map | 否 | 参数覆盖表，对工序类型默认参数的覆盖值，如 `{squeegeePressure: 120}` |
| optionalFlag | Boolean | 否 | 可选标志，默认 `false`；为 `true` 时允许跳过该步骤（如 SPI 在低密度板时可选） |
| stationConstraint | String(32) | 否 | 工位约束编码，限制该步骤必须在指定工位/设备类型上执行 |
| dataSchema | String(64) | 否 | 绑定的数据采集Schema编码，如 `SCHEMA.SMT.PRINT` |
| estimatedCycleTime | Integer | 否 | 预估节拍时间（秒），用于产能规划 |
| setupTime | Integer | 否 | 换线准备时间（分钟），用于排产计算 |

**步骤实例与工序类型的关系：**

工序类型定义"做什么"（能力模板），步骤实例定义"在哪里做、按什么参数做、前后做什么"（路线上下文）。这种分离设计使同一工序类型可复用于不同路线、不同位置，而参数覆盖机制保证了灵活性。

## 7.3 路线图语义

路线图（StepGraph）描述步骤实例之间的流转逻辑。系统支持五种基本结构，复杂路线由这些结构组合而成。

### 7.3.1 线性序列（Linear Sequence）

最基础的流转结构，步骤严格按先后顺序执行。适用于无分支的标准流水线。

```
┌──────┐    ┌──────┐    ┌──────┐    ┌──────┐
│  S1  │───>│  S2  │───>│  S3  │───>│  S4  │
│ 印刷 │    │ SPI  │    │ 贴片 │    │回流焊│
└──────┘    └──────┘    └──────┘    └──────┘
```

**约束：** 每个步骤最多一个前驱、一个后继；序列中间不可插入其他步骤。

### 7.3.2 条件分支（Conditional Branch）

根据质量检测结果决定流转方向：合格（PASS）进入下一工序，不合格（FAIL）进入返修分支。

```
                    ┌──────────────────────────────┐
                    │         返修入口              │
                    │    S_RWK_ANALYZE             │
                    │       缺陷分析                │
                    │          │                   │
                    │          v                   │
                    │    S_RWK_EXECUTE             │
                    │       返修作业                │
                    │          │                   │
                    │          v                   │
                    │    S_RWK_VERIFY              │
                    │       返修验证                │
                    │          │                   │
                    │   ┌──────┴──────┐            │
                    │   │ PASS   FAIL │            │
                    │   v            v             │
                    │  返回主线    二次返修         │
                    └──────────────┬───────────────┘
                                   ^
                                   │ (FAIL)
┌──────┐    ┌──────┐    ┌──────┐   │    ┌──────┐
│ S{N} │───>│ S{N+1}│───>│ AOI  │───┼───>│ S{N+2}│
│ 贴片 │    │回流焊 │    │ 检测 │   │    │ 目检  │
└──────┘    └──────┘    └──────┘   │    └──────┘
                         (PASS)────┘
```

**约束：** 条件分支必须绑定质量门禁（`boundQualityGate`）；门禁结果由设备/人工判定后回写系统。

### 7.3.3 可选跳过（Optional Skip）

标记为 `optionalFlag = true` 的步骤，可在工单级别决定是否执行。典型场景：SPI 步骤在低密度/成熟产品时可跳过。

```
┌──────┐    ┌──────────┐    ┌──────┐
│  S1  │───>│  S2(可选) │───>│  S3  │
│ 印刷 │    │   SPI    │    │ 贴片 │
└──────┘    └──────────┘    └──────┘
                 │
                 │ (跳过时)
                 └──────────────────>│
```

**约束：** 可选步骤跳过时，其前驱直接连接其后继；跳过决策需在工单下发前确定，运行中不可动态跳过。

### 7.3.4 并行分支（Parallel Fork）

当拼板被分割为多块单板后，各单板可并行进入后续工序。并行分支的起点通常是"分割拼板"步骤。

```
                              ┌──────┐    ┌──────┐
                         ┌───>│ 板1  │───>│ 板1  │───┐
                         │    │插件  │    │焊接  │   │
                         │    └──────┘    └──────┘   │
┌──────┐    ┌──────┐    │                          │    ┌──────┐
│ S{N} │───>│ 分板 │───>├──────┐    ┌──────┐       ├───>│ 合并  │
│ 回流 │    │      │    │ 板2  │───>│ 板2  │       │    │ 检验  │
└──────┘    └──────┘    │ 插件 │    │焊接  │       │    └──────┘
                         │      │    │      │       │
                         │    └──────┘    └──────┘   │
                         │                          │
                         │    ┌──────┐    ┌──────┐   │
                         └───>│ 板N  │───>│ 板N  │───┘
                              │插件  │    │焊接  │
                              └──────┘    └──────┘
```

**约束：** 并行分支数量由分板数量决定；每个分支独立追踪工单子任务；各分支可异步完成。

### 7.3.5 合并点（Merge Point）

多个并行分支在指定步骤汇合。汇合步骤等待所有入边分支完成后方可启动。

```
┌──────┐
│ 返修  │───┐
│ 分支1 │   │
└──────┘   │    ┌──────────┐
           ├───>│  S_MERGE │───> 主线继续
┌──────┐   │    │ 返修验证  │
│ 返修  │───┘    │  合并点  │
│ 分支2 │        └──────────┘
└──────┘
```

**约束：** 合并步骤的 `predecessors` 列表包含所有上游分支的末步骤；合并策略为 AND-join（等待全部完成）；超时未完成的分支触发异常处理流程。

## 7.4 路线版本管理

工艺路线随产品迭代、工艺优化、设备更新而持续演进，严格的版本管理是确保生产一致性和可追溯性的基础。

### 规则1：版本号规则

采用语义化版本号 `MAJOR.MINOR.PATCH`：

| 变更类型 | 版本号变化 | 典型场景 |
|---------|-----------|---------|
| 主版本（MAJOR） | +1，MINOR和PATCH归零 | 增减步骤、改变步骤顺序、变更门禁策略 |
| 次版本（MINOR） | +1，PATCH归零 | 调整参数默认值、替换等效工序类型、修改数据采集Schema |
| 修订版（PATCH） | +1 | 修正描述文字、调整预估节拍时间、更新备注 |

### 规则2：版本生效

- 新版本审批通过后，状态变为 `RELEASED`，旧版本自动变为 `OBSOLETE`
- 支持延迟生效：设置 `effectiveDate` 为未来时间，到期自动切换
- 同一产品族同时仅允许一个 `RELEASED` 版本

### 规则3：在制品兼容

- 已开工的工单继续使用开工时的路线版本，不受新版本发布影响
- 工单上记录 `routeVersion` 字段，确保全生命周期可追溯
- 返修工单使用原工单的路线版本，而非当前最新版本

### 规则4：版本追溯

- 路线定义表保留所有历史版本（包括 `OBSOLETE` 状态），不做物理删除
- 工单→路线版本→步骤实例→工序类型，形成完整的追溯链
- 质量异常回溯时，可精确还原当时的生产路径和参数配置

### 规则5：版本归档

- `OBSOLETE` 超过保留期（默认24个月）的版本可归档至冷存储
- 归档前需确认无关联的未关闭工单
- 归档版本仅保留只读查询能力，不可恢复为活跃状态

- 测试阶段ICT→FCT顺序执行，FCT依赖ICT通过

## 8.4 模板4：SMT+选择性焊接（ROUTE.PCBA.SMT_SELECTIVE.001）

### 适用场景

- 双面SMT回流后，有少量THT元件但不宜过波峰焊的产品

| 单板测试时间 | min | ≤10 | 3-15 |
| 通信接口测试覆盖率 | % | 100 | 100 |
| 测试环境温度 | ℃ | 25±5 | 20-30 |

**数据采集：** SCHEMA.TEST.FCT — 采集全部测试项结果、实测值、测试时间、测试程序版本、环境温度。

### S25 固件烧录

| 属性 | 值 |
|------|-----|
| 实例ID | S25 |
| 工序类型 | STEP_TYPE.FINISH.FLASH |
| 执行面 | NA |
| 前驱步骤 | S24 |
| 后继步骤 | S26 |
| 绑定门禁 | - |
| 可选标志 | 否 |

**关键控制点：**

1. 固件版本校验：烧录前系统校验固件文件MD5/SHA256哈希值，确保固件未被篡改且与产品BOM指定的版本一致
2. 烧录接口：根据MCU/SoC类型选择接口（SWD/JTAG/UART/USB），烧录时序和电压须严格匹配芯片规格书
3. 烧录验证：烧录完成后执行回读校验（Read-back Verify），逐字节比对，不一致即判定烧录失败
4. 熔丝/加密位：烧录完成后根据安全策略设置加密位（Lock Bit）或写保护位，防止固件被读出或篡改
5. 序列号写入：将PCBA的唯一序列号（UID）烧录至OTP区域或指定Flash地址，建立硬件与固件的绑定关系

| 烧录接口 | - | SWD（ARM） | 按芯片型号 |
| 烧录速度 | MHz | 4 | 1-10 |
| 回读校验 | - | 逐字节比对 | 逐字节/逐块 |
| 烧录重试次数 | 次 | 3 | 1-3 |

**数据采集：** SCHEMA.FINISH.FLASH — 采集固件版本号、固件哈希值、烧录时间、烧录结果（成功/失败/重试次数）、芯片UID、烧录器编号。

### S26 终检

| 属性 | 值 |

**关键控制点：**

1. 出货前最终检验：在ICT和FCT通过后、包装前，对PCBA进行出货质量确认
2. 检验项目：外观复查（运输/测试过程中是否有新增划伤或碰撞）、标签信息核对（UID、MAC地址、版本号）、附件完整性（散热片、屏蔽罩、绝缘垫片是否齐全）
3. 出货报告审核：确认该板的IQC、SMT AOI、ICT、FCT、烧录所有报告均为Pass状态，形成完整质量档案
4. 特殊要求核对：客户特殊要求（如特定固件配置、MAC地址段、序列号段）逐项核对
5. 门禁逻辑：QG-FINAL-01为出货最终门禁，关联该板全流程所有门禁记录，任一环节有未关闭的异常即阻断出货

**数据采集：** 终检通过MES终端确认，系统自动汇总该板全流程质量数据并生成出货报告（COA: Certificate of Analysis）。
