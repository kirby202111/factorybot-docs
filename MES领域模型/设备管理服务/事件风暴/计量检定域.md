# 计量检定域（Calibration Bounded Context）

> **限界上下文**：计量检定域
> **统一语言**：CalibrationRule（检定规则）, CalibrationPlan（检定计划）, CalibrationTask（检定任务）, CalibrationDispatch（送检派工）, CalibrationCertificate（检定证书）, CalibrationResult（检定结果）, ExternalLab（外部检定机构）, InstrumentStatus（计量器具检定状态）, CalibrationHistory（检定历史）
> **核心关注**：以"事件风暴"方式刻画**计量器具**从检定规则定义、周期检定计划、送检/收回流程、证书管理到过期预警的完整流程。本域负责"**何时该检、怎么检、证书怎么管**"；**不负责**计量器具的身份数据与可用性状态——那归台账域。

---

## 0. 设计原则与边界澄清

计量检定域是设备管理服务下的**计量管理子域**，是 PCBA + 整机组装混合型车间所有"计量器具检定活动"的管理权威源。

**域内职责**（本域负责）：
- 检定**规则定义**——检定周期、检定方式（内检/外检）、检定项目、外部机构绑定
- 检定**计划生成**——根据规则与证书有效期，自动生成周期检定计划
- **送检/收回流程**——计量器具送检（暂停使用）、检定执行、收回（恢复使用）
- **证书管理**——证书签发、有效期维护、过期检测
- **过期预警**——证书即将过期/已过期的预警与通知

**域外职责**（本域**不**负责）：
- 计量器具身份、归属、状态 → 台账域
- 设备/工装的点检/保养 → 点检保养域
- 故障报修/维修 → 维修域
- 设备通信与数据采集 → 接入域
- 移动端 UI 交互 → 应用层

**为什么计量检定不放点检保养域**：
1. **法规属性不同**——计量检定受国家计量法规强制约束，有法律效力；点检保养是内部管理行为，无强制法规。两者合规要求完全不同。
2. **流程特征不同**——检定涉及外部实验室、法定证书、追溯链；点检保养是纯内部活动。送检/收回流程是检定域独有的。
3. **证书管理独有**——检定证书有唯一编号、法定有效期、签发机构，是检定域的核心聚合，点检保养域无此概念。
4. **判定结果的法律含义不同**——检定不合格意味着"法定不可用"，必须立即停用；点检 FAIL 是内部判定，处理流程更灵活。

**为什么计量检定不放台账域**：
1. **职责分离**——台账域关注"资产是谁、归谁、能不能用"，检定域关注"检定活动怎么管"。混在一起台账域变成"台账+计量调度"混合体。
2. **生命周期节奏不同**——资产生命周期以年为单位，检定周期以月为单位，变更频率差异大。
3. **与台账域已有设计一致**——台账域 §10 已明确"检定周期设定、送检/收回流程、证书签发/过期判定全部归计量检定域"。

## 1. 计量器具与检定规则

### 1.1 计量器具的特殊管理要求

在 PCBA + 整机组装混合型车间，计量器具具有以下特殊性：

| 特征 | 说明 | 管理含义 |
|------|------|---------|
| **强制周期检定** | 国家计量法规定必须按周期送检 | 到期必须送检，不可跳过 |
| **证书有效期** | 每次检定签发证书，有法定有效期 | 超期即不可使用，必须冻结 |
| **内检 vs 外检** | 部分器具可内部检定，部分必须外检 | 流程分支：内检在厂完成，外检需送出 |
| **ABC 分类** | A 类（强制检定）、B 类（依法管理）、C 类（一次性/免检） | 不同分类管理严格度不同 |
| **追溯链** | 检定用标准器本身也需检定 | 形成检定溯源链 |

### 1.2 检定规则模型

```
CalibrationRule
├── rule_id              : 规则唯一标识
├── target_classification: 计量器具分类（如"卡尺""扭矩扳手""温度计"）
├── abc_category         : A | B | C
├── calibration_cycle    : 检定周期（月）
├── calibration_method   : INTERNAL（内检） | EXTERNAL（外检）
├── external_lab_id      : 外部检定机构（外检时必填）
├── calibration_items[]  : 检定项目模板
│   ├── item_name        : 项目名称
│   ├── standard_value   : 标准值
│   ├── tolerance        : 允差
│   └── measurement_unit : 单位
├── certificate_template : 证书模板（内检时）
├── lead_time_days       : 送检提前期（天）——证书到期前多久开始送检
├── responsible_role     : 负责角色
└── enabled              : 是否启用
```

**ABC 分类管理策略**：

| 分类 | 检定方式 | 证书管理 | 超期处理 |
|------|---------|---------|---------|
| A 类（强制检定） | 必须外检到法定机构 | 法定证书，有效期严格 | 超期立即冻结，法规违规告警 |
| B 类（依法管理） | 可内检或外检 | 内部证书或外部证书 | 超期冻结，无法规违规告警 |
| C 类（免检/一次性） | 首件检定后免周期检定 | 仅首次证书 | 无周期检定要求 |

## 3. 事件风暴 — 送检与检定执行

### 3.1 送检（暂停使用）

```
CalibrationTaskCreated
        |
        v
【命令:送检 DispatchCalibration】--> 【聚合:CalibrationTask】
                                        |
                                        v
                                【事件:已送检 CalibrationDispatched】
                                 (task_id, instrument_id, dispatch_method: INTERNAL/EXTERNAL,
                                  dispatched_at, expected_return_date)
                                        |
                                        v
                                发布到 calibration.dispatch → 台账域消费：SuspendAsset(reason=Calibrating)
```

> **关键**：送检意味着计量器具**暂停使用**。本域发布 `CalibrationDispatched`，台账域消费后执行 SuspendAsset。外检时器具物理离开车间，内检时器具在车间内但标记为"检定中"。

### 3.2 内检执行

```
[参与者:计量员/内检员]
        --【命令:执行内检 PerformInternalCalibration】--> 【聚合:CalibrationTask】
                                                            |
                                                            v
                                                    【事件:内检已执行 InternalCalibrationPerformed】
                                                     (task_id, instrument_id, items[], results[])
                                                            |
                                                ┌──── 检定结果 ────┐
                                                v                   v
                                            PASS                FAIL
                                                |                   |
                                                v                   v
                                    【命令:签发内检证书         【命令:记录检定不合格
                                     IssueInternalCertificate】  RecordCalibrationFailed】
                                                |                   |
                                                v                   v
                                    【事件:检定证书已签发          【事件:检定不合格
                                     CalibrationCertificateIssued  CalibrationFailed】
                                     (cert_no, instrument_id,      (instrument_id, task_id,
                                      valid_until, issued_by,       failed_items[],
                                      issued_at)                    repair_feasibility: RECALIBRATABLE | NOT_ECONOMICAL | NOT_REPAIRABLE)
                                                |                        |
                                                v                    ┌──── repair_feasibility ────┐
                                    台账域消费：ReinstateAsset       v                v                   v
                                    + UpdateCertValidity     RECALIBRATABLE   NOT_ECONOMICAL    NOT_REPAIRABLE
                                                                 |                |                   |
                                                            可重新调校       台账域消费：       台账域消费：
                                                            安排下次检定      SuspendAsset       SuspendAsset
                                                                             +建议报废           +建议报废
```

### 3.3 外检执行

```
CalibrationDispatched(method=EXTERNAL)
        |
        v
【外部系统:外部检定机构】
        -- 执行检定（域外黑盒）
        |
        v
[参与者:计量管理员]
        --【命令:登记外检结果 RecordExternalCalibrationResult】--> 【聚合:CalibrationTask】
                                                                    |
                                                        ┌──── 结果 ────┐
                                                        v               v
                                                    PASS            FAIL
                                                        |               |
                                                        v               v
                                        【命令:录入外部证书          【命令:记录外检不合格
                                         RegisterExternalCertificate】 RecordExternalCalibrationFailed】
                                                        |               |
                                                        v               v
                                        【事件:检定证书已签发          【事件:检定不合格
                                         CalibrationCertificateIssued  CalibrationFailed】
                                         (cert_no, instrument_id,        (instrument_id, task_id,
                                          valid_until, issued_by,         failed_items[],
                                          issued_at, external_lab_id)      external_lab_id,
                                                                           repair_feasibility)
                                                        |                   |
                                                        v                   v
                                        台账域消费：                  台账域消费：
                                        ReinstateAsset               SuspendAsset(reason=CalibrationFailed)
                                        + UpdateCertValidity         repair_feasibility 为 NOT_ECONOMICAL 或
                                                                     NOT_REPAIRABLE 时，台账域可发起报废审批
```

### 3.4 收回

```
CalibrationCertificateIssued / CalibrationFailed
        |
        v
【命令:收回器具 ReceiveCalibratedInstrument】--> 【聚合:CalibrationTask】
                                                    |
                                                    v
                                            【事件:检定器具已收回 InstrumentCalibrationReceived】
                                             (task_id, instrument_id, received_at, received_by)
                                                    |
                                            仅对 PASS 的情况：
                                            台账域已通过 CalibrationCertificateIssued 消费
                                            执行了 ReinstateAsset，收回事件为补充记录
```

## 4. 事件风暴 — 证书管理与过期预警

### 4.1 证书生命周期

```
CalibrationCertificateIssued
        |
        v
【聚合:CalibrationCertificate】
        |
        v
    [VALID] ──(valid_until - lead_time_days ≤ now)──► [EXPIRING_SOON] ──(valid_until ≤ now)──► [EXPIRED]
                     |                                           |
                     v                                           v
            【事件:证书即将过期                                 【事件:证书已过期
             CalibrationExpiringSoon】                          CalibrationExpired】
             (instrument_id, cert_no, valid_until,              (instrument_id, cert_no,
              days_remaining)                                    expired_since)
                     |                                           |
                     v                                           v
            通知工程团队                                     台账域消费：SuspendAsset(reason=CalibrationExpired)
            （不触发状态变更）                               + 本域自动触发 CalibrationDue
```

### 4.2 证书模型

```
CalibrationCertificate
├── cert_no         : 证书编号（全局唯一）
├── instrument_id   : 计量器具 ID
├── issued_by       : 签发机构（内部/外部实验室名称）
├── issued_at       : 签发日期
├── valid_from      : 有效起始日期
├── valid_until     : 有效截止日期
├── calibration_result: PASS | FAIL（仅 PASS 才签发证书）
├── calibration_items[]: 检定项目与结果
├── external_lab_id : 外部实验室 ID（外检时）
├── status          : VALID | EXPIRING_SOON | EXPIRED | REVOKED
└── attachment_urls[]: 证书扫描件/电子证书
```

> 🔴 **热点：证书编号规则**——内检证书与外检证书是否统一编号？建议：**统一前缀区分**——内检 `CI-YYYY-NNNNN`，外检 `CE-YYYY-NNNNN`，便于区分但统一管理。

### 4.3 过期预警策略

| 预警级别 | 触发条件 | 动作 |
|---------|---------|------|
| 预警（黄色） | valid_until - 30 天 ≤ now | 通知计量管理员，准备送检 |
| 临期（橙色） | valid_until - lead_time_days ≤ now | 自动触发 CalibrationDue，生成检定计划 |
| 过期（红色） | valid_until ≤ now | 发布 CalibrationExpired，台账域冻结资产；自动触发紧急检定 |

## 5. 聚合与不变式

| 聚合 | 一致性边界 | 不变式 |
|------|-----------|--------|
| **CalibrationRule** | 单条检定规则 | 同一 `target_classification` 不重复；检定周期 > 0；外检时 external_lab_id 必填 |
| **CalibrationPlan** | 单个周期检定计划 | 每条 plan_item 必须关联已激活的 CalibrationRule；plan 内 instrument 不重复 |
| **CalibrationTask** | 单次检定任务 | 必须关联 CalibrationPlan 的 plan_item；送检后必须记录结果才能收回；PASS 必须签发证书 |
| **CalibrationCertificate** | 单张检定证书 | cert_no 全局唯一；签发时 calibration_result 必须为 PASS；valid_until > issued_at |

## 6. 关键策略（Policy）汇总

### 6.1 触发策略

| 触发源 | 策略 | 触发命令 |
|--------|------|----------|
| 台账域 `InstrumentCertValidityUpdated` | 证书有效期与提前期比较 | MarkCalibrationDue（临期时） |
| 日历定时器 | 周期扫描检定规则 | MarkCalibrationDue（达到周期时） |
| CalibrationExpired | 过期自动触发紧急检定 | MarkCalibrationDue(due_reason=CertExpired) |

### 6.2 调度策略

| 触发事件 | 策略 | 触发命令 |
|----------|------|----------|
| CalibrationDue | 自动生成检定计划 | GenerateCalibrationPlan |
| CalibrationPlanGenerated | 拆解为检定任务 | CreateCalibrationTask |

### 6.3 结果策略

| 触发事件 | 策略 | 触发命令 |
|----------|------|----------|
| InternalCalibrationPerformed(result=PASS) | 签发内检证书 | IssueInternalCertificate |
| InternalCalibrationPerformed(result=FAIL) | 记录检定不合格（含处置判定） | RecordCalibrationFailed（携带 repair_feasibility） |
| 外检 PASS | 录入外部证书 | RegisterExternalCertificate |
| 外检 FAIL | 记录外检不合格（含处置判定） | RecordExternalCalibrationFailed（携带 repair_feasibility） |
| CalibrationCertificateIssued | 通知台账域更新证书有效期 + 恢复资产 | （通过事件契约，台账域消费） |
| CalibrationFailed | 通知台账域冻结资产 | （通过事件契约，台账域消费） |
| CalibrationExpired | 通知台账域冻结资产 + 自动触发紧急检定 | MarkCalibrationDue(due_reason=CertExpired) |

### 6.4 资产生命周期联动

| 触发源（台账域事件） | 策略 |
|---------------------|------|
| AssetCommissioned（asset_kind=INSTRUMENT） | 激活该器具匹配的检定规则 |
| AssetRetired | 停用该器具检定规则，取消未执行任务 |

## 8. 与台账域的协作边界

| 维度 | 本域（计量检定域） | 台账域 |
|------|-------------------|--------|
| 关心什么 | 检定活动**怎么管**、证书**怎么发** | 资产**能不能用** |
| 存储什么 | 检定规则、检定周期、证书全量数据 | cert_valid_until（仅当前值） |
| 谁判定过期 | **本域判定**——证书有效期归本域管理 | **本域消费**——收到 CalibrationExpired 后冻结 |

**事件流向**：

```
本域 ──→ calibration.dispatch ──→ 台账域（SuspendAsset(reason=Calibrating)）
本域 ──→ calibration.result(PASS) ──→ 台账域（ReinstateAsset + UpdateCertValidity）
本域 ──→ calibration.result(FAIL) ──→ 台账域（SuspendAsset(reason=CalibrationFailed)）
本域 ──→ calibration.result(FAIL, feasibility=NOT_ECONOMICAL/NOT_REPAIRABLE) ──→ 台账域（可发起报废审批）
本域 ──→ calibration.expiry ──→ 台账域（CalibrationExpired→SuspendAsset）
台账域 ──→ eam.asset.metric(CertValidityUpdated) ──→ 本域（辅助触发判定）
台账域 ──→ eam.asset.lifecycle ──→ 本域（Commissioned→激活规则 / Retired→停用规则）
```

> **关于证书有效期事件中转的设计说明**：本域签发证书后自身拥有 `valid_until` 数据，但仍订阅台账域的 `InstrumentCertValidityUpdated` 事件。设计意图为**双重保障**：(1) 台账域可能通过其他途径更新 `cert_valid_until`（如批量导入、ERP 同步），本域需感知这些变更；(2) 证书过期检测由本域的定时器独立执行，不依赖台账域事件触发，`InstrumentCertValidityUpdated` 仅作为补充校验。主触发逻辑仍使用本域自身的证书数据。
