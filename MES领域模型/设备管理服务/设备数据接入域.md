# PCBA 数据采集域（Data Collection Bounded Context）

> **限界上下文**：数据采集域  
> **统一语言**：DataSchema, CollectionPoint, TraceabilityModel, DataQuality, CollectionProtocol, EdgeGateway  
> **核心关注**：定义四层采集架构、设备/事件/环境数据采集Schema、数据质量管控规则和数据存储策略  
> **原始章节**：第5章（数据采集模型）

---

## 领域概述

数据采集域是 PCBA MES 系统的**支撑域**，覆盖从设备层到应用层的完整数据采集架构。本域定义了五大类数据（设备状态、工艺参数、质量检测、生产事件、环境数据）的采集规范、协议接口、质量管控规则和存储策略，确保从生产现场到MES平台的数据贯通。

**跨领域协作**：
- 本域为 **工序域** 的每个工序类型提供数据采集Schema定义
- 本域为 **质量域** 的检验方法提供检测数据的采集和存储
- 本域为 **返修域** 的返修作业提供数据记录
- 本域为 **物料域** 的物料事件提供上料/退料数据采集
- 本域的设备数据采集与 **系统集成域** 的设备集成接口紧密协作
- 本域的数据存储为 **Agent域** 的知识检索提供数据基础

# 数据采集模型

## 5.1 数据采集架构

本章定义车间 MES 系统中数据采集的层次化架构，涵盖设备层、边缘层、平台层与应用层的采集策略、数据点定义及采集规范。数据采集模型面向电子制造与整机组装混合型车间的实际设备与系统环境，确保从生产现场到 MES 平台的数据贯通。

### 5.1.1 采集层次架构

```
┌─────────────────────────────────────────────────────────────────────────┐
│                          应用层（Application Layer）                      │
│    MES应用    │  SPC分析    │  质量追溯    │  设备OEE    │  Agent知识库    │
├─────────────────────────────────────────────────────────────────────────┤
│                          平台层（Platform Layer）                         │
│   时序数据库   │  关系数据库   │  消息队列    │  数据清洗   │  数据路由     │
├─────────────────────────────────────────────────────────────────────────┤
│                          边缘层（Edge Layer）                             │
│   Edge Gateway │  协议适配   │  本地缓存    │  断点续传   │  数据脱敏     │
├─────────────────────────────────────────────────────────────────────────┤
│                          设备层（Device Layer）                           │
│   贴片机   │  回流焊   │  AOI/SPI  │  ICT/FCT  │  印刷机   │  PLC/DCS    │
└─────────────────────────────────────────────────────────────────────────┘
```

**各层职责：**

| 层次 | 职责 | 关键组件 | 数据流向 |
|------|------|----------|----------|
| 设备层 | 原始数据产生，设备运行状态、工艺参数、检测结果的实时输出 | 生产设备、检测设备、传感器、PLC | 上行：设备 → 边缘层 |
| 边缘层 | 协议转换、数据缓存、初步清洗、断点续传保障 | Edge Gateway、协议适配器 | 上行：边缘层 → 平台层；下行：平台层 → 设备层（指令下发） |
| 平台层 | 数据持久化、数据清洗与标准化、数据路由分发、消息队列缓冲 | 时序数据库、关系数据库、Kafka/RabbitMQ | 上行：平台层 → 应用层；存储：平台层 → 数据库 |
| 应用层 | 业务数据消费、分析报表、决策支持、Agent 知识检索 | MES 模块、SPC、OEE、追溯、Agent | 消费：应用层 ← 平台层 |

## 5.2 数据分类体系

### 5.2.1 数据大类

根据数据来源与业务用途，车间数据分为五大类：

| 数据大类 | 编码 | 说明 | 典型数据项 | 采集频率 |
|----------|------|------|-----------|----------|
| 设备状态数据 | DATA-MC | 设备运行状态、报警、启停事件 | 运行/待机/故障/关机状态、报警代码、转速、温度 | 秒级 / 事件触发 |
| 工艺参数数据 | DATA-PR | 工序执行过程中的关键工艺参数 | 炉温曲线、印刷压力、贴装速度、焊接温度 | 秒级 / 批次级 |
| 质量检测数据 | DATA-QC | AOI、SPI、ICT、FCT、X-Ray、目检等检测结果 | 缺陷编码、检测值、判定结果、检测图像索引 | 单件级 |
| 生产事件数据 | DATA-EV | 生产过程中的事务性事件 | 过站记录、工单开始/结束、物料上料/退料、异常事件 | 事件触发 |
| 环境数据 | DATA-ENV | 车间环境监控数据 | 温湿度、ESD、洁净度、气压 | 分钟级 |

### 5.2.2 数据采集优先级矩阵

| 优先级 | 等级 | 采集要求 | 适用数据类型 | 典型场景 |
|--------|------|----------|-------------|----------|
| P0 | 强制实时 | 必须实时采集，延迟 < 1s，丢失率 < 0.01% | 设备状态数据、关键工艺参数 | 回流焊炉温、贴片机状态、AOI 判定 |
| P1 | 强制近实时 | 必须采集，延迟 < 5s，丢失率 < 0.1% | 质量检测数据、生产事件数据 | 过站记录、ICT/FCT 检测结果 |
| P2 | 强制批量 | 必须采集，允许批量上报（延迟 < 60s） | 批次级工艺参数、环境数据 | 锡膏回温记录、车间温湿度 |
| P3 | 可选采集 | 根据产线配置选择性采集 | 辅助传感器数据、能耗数据 | 设备能耗、振动监测 |

## 5.6 数据采集协议与接口

### 5.6.1 支持的采集协议

| 协议 | 适用场景 | 传输方式 | 典型设备 |
|------|---------|----------|----------|
| SECS/GEM | 半导体/SMT 设备标准通信 | TCP/IP | 贴片机、AOI、SPI（高端机型） |
| OPC-UA | 工业设备互操作标准 | TCP/IP / HTTPS | 回流焊、波峰焊、PLC |
| Modbus TCP | 工业 PLC/传感器通信 | TCP/IP | 温湿度传感器、PLC 控制器 |
| MQTT | 轻量级 IoT 数据上报 | TCP/IP（发布/订阅） | 环境传感器、边缘网关 |
| HTTP/REST | 系统间数据交互 | HTTPS | MES 与 WMS/ERP 系统集成 |
| 文件导入（CSV/XML） | 离线设备数据批量导入 | 文件传输 | 老旧设备、手动检测设备 |
| 串口（RS-232/485） | 老旧设备直连 | 串行通信 | 早期测试设备、条码扫描枪 |

### 5.6.2 边缘网关功能

边缘网关是设备层到平台层的关键桥梁，提供以下核心能力：

| 功能 | 说明 | 技术要求 |
|------|------|----------|
| 协议转换 | 将不同设备协议统一转换为 MQTT/HTTP 标准格式 | 支持 SECS/GEM、OPC-UA、Modbus、串口等协议的解析与转换 |
| 数据缓存 | 网络中断时本地缓存数据，恢复后自动补传 | 本地存储容量 ≥ 48h 数据量，支持 FIFO 覆盖策略 |
| 断点续传 | 记录已上报数据的时间戳/序号，恢复后从断点续传 | 每设备维护独立的上报游标（cursor），支持 Exactly-Once 语义 |
| 数据脱敏 | 对敏感信息进行脱敏处理（如操作员姓名） | 支持可配置的脱敏规则（掩码、哈希、替换） |
| 本地诊断 | 边缘网关自身状态监控 | CPU、内存、磁盘、网络状态、数据积压量 |
| 远程管理 | 支持平台侧远程配置、升级、重启 | 基于 MQTT 的命令下发通道 |

### 5.6.3 数据格式规范

**标准 MQTT Topic 结构：**

```
factory/{factory_id}/workshop/{workshop_id}/line/{line_id}/station/{station_id}/{data_type}
```

- 设备状态：`factory/FACT-001/workshop/WS-SMT-01/line/LINE-SMT-01/station/WS-STN-SMT-03/machine_status`

- 工艺参数：`factory/FACT-001/workshop/WS-SMT-01/line/LINE-SMT-01/station/WS-STN-SMT-06/process_param`

## 5.7 数据质量管控

### 5.7.1 数据质量维度

| 质量维度 | 定义 | 度量指标 | 目标值 |
|----------|------|----------|--------|
| 完整性 | 数据点无缺失 | 采集覆盖率 = 实际采集点数 / 应采集点数 | ≥ 99.9% |
| 准确性 | 数据值与真实值一致 | 校验通过率 = 通过校验的数据点数 / 总数据点数 | ≥ 99.5% |
| 及时性 | 数据在规定的延迟内到达 | 及时率 = 延迟达标的数据点数 / 总数据点数 | ≥ 99.9%（P0）/ ≥ 99%（P1） |
| 一致性 | 同一数据在不同系统中的值一致 | 一致率 = 一致的数据点数 / 交叉校验点数 | ≥ 99.9% |
| 唯一性 | 无重复记录 | 重复率 = 重复记录数 / 总记录数 | < 0.01% |

### 5.7.2 数据校验规则

| 校验类型 | 规则说明 | 示例 |
|----------|----------|------|
| 范围校验 | 数据值在合理物理范围内 | 回流焊温度：180℃ ≤ T ≤ 300℃ |
| 变化率校验 | 数据变化率不超过物理可能 | 温度变化率 ≤ 5℃/s |
| 逻辑校验 | 关联数据点之间的逻辑关系合理 | 设备运行时，工艺参数不能为空 |
| 时序校验 | 时间戳单调递增，无倒退 | t(n+1) ≥ t(n) |
| 存在性校验 | 必填数据点不能为空 | 过站事件中 SN 不能为 NULL |
| 枚举校验 | 数据值在预定义的枚举范围内 | 设备状态只能是 ACTIVE/IDLE/FAULT/OFFLINE |
| 关联校验 | 关联外键有效 | station_id 必须在工位主数据中存在 |

### 5.7.3 异常数据处理策略

| 异常类型 | 处理策略 | 说明 |
|----------|----------|------|
| 数据缺失 | 标记为 NULL，不填充默认值 | 避免假数据误导分析，后续由分析层处理缺失 |
| 数据越界 | 标记质量位为 SUSPECT，保留原值 | 记录越界详情，触发告警，由工艺工程师确认 |
| 时间戳异常 | 按采集到达时间修正，记录原始时间戳 | 保留审计线索 |
| 重复数据 | 按 msg_id 去重，保留首次到达数据 | 消息队列保证 Exactly-Once 投递 |
| 数据乱序 | 按 timestamp 重排序，窗口容忍 5s | 超过窗口的乱序数据标记 LATE 并单独存储 |

## 5.8 数据存储策略

### 5.8.1 存储分层

| 存储层 | 数据库类型 | 数据范围 | 保留周期 | 用途 |
|--------|-----------|----------|----------|------|
| 热数据层 | 时序数据库（TDengine / InfluxDB） | 最近 30 天设备状态、工艺参数 | 30 天 | 实时监控、实时分析 |
| 温数据层 | 关系数据库（PostgreSQL / MySQL） | 最近 1 年生产事件、质量数据 | 1 年 | 追溯查询、MES 业务 |
| 冷数据层 | 对象存储 / 数据仓库 | 1 年以上历史数据 | 3-5 年 | 长期分析、合规审计 |
| 归档层 | 离线存储（磁带 / 冷备） | 超过保留周期的数据 | 永久 | 合规归档 |

### 5.8.2 数据压缩策略

| 压缩方法 | 适用数据 | 压缩比（典型） | 说明 |
|----------|----------|---------------|------|
| 死区压缩 | 稳态设备状态数据 | 10:1 ~ 50:1 | 数据变化小于阈值时不上报新值 |
| 旋转门压缩 | 缓变工艺参数（温度、湿度） | 5:1 ~ 20:1 | 保留趋势变化的关键拐点数据 |
| 降采样 | 历史数据归档 | 10:1 ~ 100:1 | 按时间窗口聚合（avg/max/min） |
