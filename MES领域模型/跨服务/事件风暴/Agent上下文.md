# PCBA Agent域（Agent Knowledge Base Bounded Context）

> **限界上下文**：Agent域  
> **统一语言**：KnowledgeBase, QueryPattern, RetrievalStrategy, ResponseTemplate, IntentMapping  
> **核心关注**：定义Agent知识库的文档检索结构、编码/名称/语义三级检索策略、查询模式映射和标准化响应模板  

---

## 领域概述

Agent域是 PCBA MES 系统的**通用域**，为AI Agent提供跨领域的知识检索和响应框架。本域定义了三级递进式检索策略（编码精确匹配 → 实体名称匹配 → 语义搜索），覆盖所有领域的查询意图映射，以及4种标准化响应模板。

**跨领域协作**：
- 本域的检索结构覆盖 **所有其他领域** 的文档和编码体系
- 本域的查询模式映射表将用户自然语言查询路由到正确的领域和章节
- 本域的响应模板组合多个领域的信息输出结构化答案
- 本域依赖 **制造实体域** 的编码体系（SN、工单号）
- 本域依赖 **工序域** 的工序编码（STEP_TYPE.*）
- 本域依赖 **质量域** 的门禁编码（QG-*）和缺陷编码（DEF-*）
- 本域依赖 **异常处理域** 的异常编码（EX-*）
- 本域依赖 **系统集成域** 的接口编码（IF-*）

## 14.3 知识检索策略

Agent 采用三级递进式检索策略，从精确到模糊逐级降级，确保检索的准确性和覆盖度。

### 14.3.1 一级：编码精确匹配

**触发条件**：用户输入包含已知编码前缀（STEP_TYPE / QG / ROUTE / DEF / SCHEMA / EX / IF 等）。

**检索方式**：解析编码前缀和编号，在编码索引中精确定位到对应的定义章节。

**检索示例**：

| 输入 | 解析 | 定位 |
|------|------|------|
| `SMT-T-03` | 工序编码：SMT T面 第3序 | PCBA工艺流程详细设计文档 -> 阶段一 -> 锡膏印刷条目 |
| `QG-SMT-AOI-01` | 质量门禁：SMT AOI 第1号 | 第11.2章 -> QG-SMT-AOI-01 行 |
| `EX-EQP-01` | 异常编码：设备类 故障停机 | 第12.1.1章 -> 设备异常 -> EX-EQP-01 行 |
| `IF-WMS-01` | 集成接口：WMS 第1号 | 第13.1.1章 -> IF-WMS-01 行 |
| `ROUTE.PCBA.DBL_MIXED.001` | 工艺路线：PCBA 双面混装 | 第11.2章 + PCBA工艺流程详细设计文档全文 |

### 14.3.2 二级：实体名称匹配

**触发条件**：用户输入包含中英文实体名称（如"锡膏印刷"、"AOI"、"回流焊"），但未使用精确编码。

**检索方式**：在关键词索引中匹配实体名称，映射到对应的编码，再通过一级检索定位。

**检索示例**：

| 输入 | 名称匹配 | 编码映射 | 定位 |
|------|----------|----------|------|
| "锡膏印刷" | 工序名称 | STEP_TYPE.SMT.PRINT -> SMT-T-03 | 一级检索 |
| "AOI" | 设备类型 / 工序名称 | AOI -> SMT-T-07 / SMT-B-06 | 一级检索 |
| "首件目检" | 工序名称 | SMT-T-05 / SMT-B-04 / THT-03 | 一级检索（多匹配，按上下文消歧） |
| "波峰焊" | 设备类型 / 工序名称 | THT-04 | 一级检索 |
| "返修" | 流程名称 | 返修流程章节 | PCBA工艺流程详细设计文档 第九章 |

### 14.3.3 三级：语义搜索

**触发条件**：用户输入自然语言问题，不包含精确编码或标准实体名称。

**检索方式**：对问题进行分词和关键词提取，匹配到相关章节的关键词索引。可能需要跨章节组合答案。

**检索示例**：

| 输入 | 关键词提取 | 匹配章节 | 答案组合 |
|------|-----------|----------|----------|
| "AOI不合格的产品怎么处理" | AOI, 不合格, 处理 | 第11.2章（门禁失败动作）+ 第4章（返修流程）+ 第10.1章（状态转换：IN_PROCESS->REWORK） | 跨章节组合：先查门禁规则 -> 失败动作是返修 -> 查返修流程 -> 返修后状态转换回IN_PROCESS |
| "炉温偏了会有什么后果" | 炉温, 偏离, 后果 | 第12.1.1章（EX-PROC-01）+ 第12.2.2章（响应流程）+ PCBA工艺流程详细设计文档 第十二章（参数范围） | 跨章节组合：异常分类 -> 响应流程 -> 参数参考范围 |
| "怎么知道这批料能不能用" | 物料, 能不能用, 校验 | 第13.1章（库存查询接口 IF-WMS-05）+ 第12.1.1章（物料异常 EX-MATL-01~05）+ 车间过点流程.md 第四章（物料校验规则） | 跨章节组合：WMS库存查询 -> 物料校验规则 -> 异常分类 |

## 14.4 Agent 响应模板

### 14.4.1 模板1：工序查询响应

```
【工序名称】（工序编码）
类别：{阶段类别}，{工序类型}
说明：{一句话描述}

关键控制点：
- {控制点1}
- {控制点2}
- {控制点3}

质量门禁：{绑定的门禁编码} - {门禁名称}
数据采集：{Schema编码}，采集方式：{自动/人工/混合}
设备要求：{设备类型列表}
参数范围：
- {参数1}：{范围}
- {参数2}：{范围}

前置条件：{前置工序编码} 须已PASS
后置工序：{后置工序编码}

相关异常类型：{EX-xxx}
```

**示例响应**：

```
【锡膏印刷 T面】（SMT-T-03）
类别：阶段一 SMT T面贴装，锡膏印刷工序
说明：通过钢网将焊膏精确印刷至 T 面焊盘，是 SMT 工艺的首个关键工序

关键控制点：
- 印刷压力、速度、脱模参数符合工艺文件
- 锡膏回温与搅拌记录齐全
- 钢网张力与开孔状态合格

质量门禁：QG-SMT-SPI-01 - SMT锡膏印刷质量（如有SPI）
数据采集：SCHEMA.SMT-T-03，采集方式：设备联机自动采集
设备要求：锡膏印刷机、钢网、刮刀
参数范围：
- 印刷速度：20–80 mm/s
- 印刷压力：3–8 kgf
- 脱模速度：0.5–3 mm/s

前置条件：SMT-T-02（T面投板）须已PASS
后置工序：SMT-T-04（贴片T面）

相关异常类型：EX-PROC-02（印刷参数超差）
```

**示例响应**：

```
【SMT T面贴装质量】（QG-SMT-AOI-01）
触发时机：SMT-T-07 AOI T面判定，回流焊后AOI检测完成
规则表达式：
  aoi_result == "PASS"
  OR (aoi_result == "REVIEW"
      AND manual_confirm == "PASS"
      AND defect_severity != "CRITICAL")

判定方式：AOI自动判定 + 人工复核（REVIEW件）

**示例响应**：

```
【炉温曲线偏移超差】（EX-PROC-01）
严重等级：L2-严重
触发条件：回流焊炉温曲线测定发现峰值温度、保温时间等关键参数超出工艺规格

即时处置：
- 立即停线，停止该炉次进板
- 通知工艺工程师到场

遏制措施：
- 隔离该炉次产出的所有PCBA，标记待复判
- 确认偏移开始时间，追溯受影响的产品范围
- 检查后续炉次（如同一产线其他回流炉）是否有同样偏移

根因分析：
- 检查炉温设定值与实测值的偏差
- 确认测温板/热电偶是否失效（更换测温板复测）
- 检查排风/冷却系统是否正常
- 对比近期炉温曲线趋势，判断是突发还是渐变

纠正与验证：
- 调整温区参数/链速至工艺规格内
- 重新测定炉温曲线，连续3次合格
- 对隔离的PCBA逐板或抽样复判焊点质量
- 首件验证合格后复产

恢复条件：
- 炉温曲线连续测定3次合格
- 受影响PCBA焊点质量复判通过（或已返修合格）
- 工艺工程师确认签字

升级路径：
- 30分钟内未恢复 -> 升级为L1，通知生产经理
- 同一设备7天内再次发生 -> 升级为L1，启动设备深度维护

相关门禁/工序：SMT-T-06（回流焊T面）、SMT-B-05（回流焊B面）、THT-04（过波峰焊）
