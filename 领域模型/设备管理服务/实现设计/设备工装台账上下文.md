> **限界上下文**：设备工装台账
> **所属服务**：设备管理服务
> **实现依据**：[领域建模 — 设备工装台账上下文](../领域建模/设备工装台账上下文.md)；辅助追溯：[事件风暴 — 设备工装台账上下文](../事件风暴/设备工装台账上下文.md)

## 0. 实现总览

设备工装台账上下文是设备管理服务下的资产主数据与可用性权威源，代码实现以 `EquipmentModel`、`FixtureModel`、`MeasuringInstrumentModel` 三类资产聚合保护单件资产生命周期，以 `AvailabilityProjectionModel` 维护单件资产可用性投影，以 `ScrapApplicationModel` 管理报废审批，以 `FixtureMatchingRuleModel` 发布工装适用规则。实现重点是：资产状态机、硬阻塞/条件性阻塞、度量累积与基准重置、域外事件驱动可用性重算、报废审批通过后强一致退役、事件 Outbox 最终一致发布。

### 分层边界

```text
facade / controller  ->  facade-impl  ->  application  ->  domain
                                              |              ^
                                              v              |
                                        repository / infrastructure
```

**本实现坚持**：
- 领域层不依赖 Spring / MyBatis / MQ / HTTP；聚合根只表达业务状态、行为、不变式和领域事件。
- 应用服务负责编排、事务、幂等、只读加载、操作日志、事件收集与 Outbox，不写仓储，不写 INV 业务判断。
- 领域服务负责跨聚合规则、加载聚合、调用聚合行为、写仓储持久化，并返回领域事件。
- 仓储层用 MyBatis 装配写模型聚合；基础设施层实现 Outbox、MQ Listener、ACL、ERP/PLM/财务适配。
- 读侧围绕台账列表、可用资产候选、规格画像、工装匹配规则、报废申请查询组织投影，不反向污染写模型聚合。

| 层 | 职责 | 禁止事项 |
|----|------|----------|

| `facade-impl` | 参数校验、DTO/Request ↔ Info 转换、调用 AppService、输出 DTO | 禁止承载 INV 判断；禁止直接调 Mapper / RepositoryImpl |
| `controller` | HTTP 入口，如资产登记、报废审批、可用资产查询 | 只能调 facade 声明方法；禁止写业务规则 |
| `application` | 用例编排、事务、幂等、只读查询、事件消费入口、Outbox 写入、操作日志 | 禁止直接调用 Repository 写方法；禁止把状态机写成 if/else 流水账 |
| `domain` | 聚合根、值对象、领域事件、领域服务、Repository/Client/FacadeClient 接口 | 禁止 Spring / MyBatis / MQ / HTTP 注解；禁止依赖外部 DTO |
| `repository` | RepositoryImpl、Mapper、DO、Model↔DO Converter、读模型 SQL | 禁止 DO 上抛；禁止绕过聚合修改核心状态 |
| `infrastructure` | MQ、Outbox、ERP/PLM/财务/设备接入 ACL 实现与配置 | 禁止承载资产状态机、报废审批等业务规则 |
| `utility` | 无业务语义工具类、统一异常基类、时间接口 | 禁止放置资产业务判断 |

### 2.1 EquipmentModel（设备）

**来源**：领域建模 §1.1
**一致性边界**：单台设备从入账到报废的全生命周期状态、位置归属、采集通道绑定与运行时长累积。
**实现形态**：`domain.model.equipment.EquipmentModel`（充血聚合根）

```java
public class EquipmentModel {
    private AssetIdVO assetId;
    private AssetIdentifierVO identifier;
    private String model;
    private String vendor;
    private AssetClassificationVO classification;
    private EquipmentSpecificationProfileVO specificationProfile;
    private AvailabilityStateVO availabilityState;
    private AssetLocationVO location;
    private String custodianOrg;
    private String custodianPerson;
    private StationReferenceVO stationId;
    private ChannelReferenceVO dataChannelId;
    private RunHourAccrualVO runHourAccrual;
    private List<BlockingReasonVO> blockingReasons;
    private Long version;
    private final List<Object> domainEvents = new ArrayList<>();

    public void commission() {
        requireNotRetired();
        requireState(AvailabilityStateVO.NEW);
        requireSpecificationProfileEstablished();
        this.availabilityState = AvailabilityStateVO.IN_SERVICE;
        registerEvent(new AssetCommissionedEvent(assetId));
    }

    public void suspend(BlockingReasonVO reason) {
        requireNotRetired();
        requireState(AvailabilityStateVO.IN_SERVICE);
        reason.requireHardBlocking();
        this.availabilityState = AvailabilityStateVO.SUSPENDED;
        this.blockingReasons = BlockingReasonVO.appendDistinct(blockingReasons, reason);
        registerEvent(new AssetSuspendedEvent(assetId, reason));
    }

    public void addBlockingReason(BlockingReasonVO reason) {
        requireNotRetired();
        requireState(AvailabilityStateVO.SUSPENDED);
        reason.requireHardBlocking();
        this.blockingReasons = BlockingReasonVO.appendDistinct(blockingReasons, reason);
    }

    public void reinstate() {
        requireNotRetired();
        requireState(AvailabilityStateVO.SUSPENDED);
        requireNoHardBlockingReasons();
        this.availabilityState = AvailabilityStateVO.IN_SERVICE;
        registerEvent(new AssetReinstatedEvent(assetId));
    }

    public void retire() {
        requireNotRetired();
        this.availabilityState = AvailabilityStateVO.RETIRED;
        registerEvent(new AssetRetiredEvent(assetId));
    }

    public void accrueRunHours(RunHourDeltaVO delta, MetricSourceVO source) {
        requireNotRetired();
        this.runHourAccrual = this.runHourAccrual.accrue(delta);
        registerEvent(new EquipmentRunHourAccruedEvent(assetId, delta, runHourAccrual, source));
    }
}
```

### 2.2 FixtureModel（工装夹具）

**来源**：领域建模 §1.2
**一致性边界**：单件工装从入账到报废的生命周期状态、借出归还、清洁翻修与使用次数累积。
**实现形态**：`domain.model.fixture.FixtureModel`（充血聚合根）

```java
public class FixtureModel {
    private AssetIdVO assetId;
    private AssetIdentifierVO identifier;
    private String model;
    private String vendor;
    private AssetClassificationVO classification;
    private FixtureSpecificationProfileVO specificationProfile;
    private AvailabilityStateVO availabilityState;
    private AssetLocationVO location;
    private String custodianOrg;
    private String custodianPerson;
    private IssuanceRecordVO issuedTo;
    private CycleAccrualVO cycleAccrual;
    private List<BlockingReasonVO> blockingReasons;
    private Long version;
    private final List<Object> domainEvents = new ArrayList<>();

    public void issueToStation(StationReferenceVO stationId, String issuedTo, DateTimeVO expectedReturn) {
        requireNotRetired();
        requireState(AvailabilityStateVO.IN_SERVICE);
        requireNotIssued();
        this.issuedTo = IssuanceRecordVO.create(stationId, issuedTo, expectedReturn);
        registerEvent(new FixtureIssuedToStationEvent(assetId, this.issuedTo));
    }

    public void returnToStorage(FixtureConditionVO condition) {
        requireNotRetired();
        requireIssued();
        this.issuedTo = null;
        registerEvent(new FixtureReturnedToStorageEvent(assetId, condition));
    }

    public void refurbish(RefurbishLogVO log, boolean lifeReset) {
        requireNotRetired();
        if (lifeReset) {
            this.cycleAccrual = CycleAccrualVO.zero();
        }
        registerEvent(new FixtureRefurbishedEvent(assetId, log, lifeReset, cycleAccrual.resetBaseline()));
    }
}
```

### 2.4 AvailabilityProjectionModel（可用性投影）

**来源**：领域建模 §1.4
**一致性边界**：单件资产当前可用性判定结果及全部阻塞原因。
**实现形态**：`domain.model.availability.AvailabilityProjectionModel`（投影聚合根）

```java
public class AvailabilityProjectionModel {
    private AssetIdVO assetId;
    private boolean available;
    private List<BlockingReasonVO> blockingReasons;
    private boolean lifecycleBlocked;
    private boolean inspectionOverdue;
    private boolean maintenanceOverdue;
    private boolean calibrationBlocked;
    private boolean malfunctionActive;
    private boolean lifeExceeded;
    private boolean commLost;
    private boolean criticalAlarm;
    private Long version;
    private final List<Object> domainEvents = new ArrayList<>();

    public void markUnavailable(BlockingReasonVO reason) {
        this.blockingReasons = BlockingReasonVO.appendDistinct(blockingReasons, reason);
        setDimension(reason, true);
        recompute(reason.toTriggerEvent());
    }

    public void clearBlocking(BlockingReasonTypeVO reasonType) {
        this.blockingReasons = BlockingReasonVO.removeByType(blockingReasons, reasonType);
        setDimension(reasonType, false);
        recompute(reasonType.toTriggerEvent());
    }

    public void clearBlockingByCategory(BlockingReasonCategoryVO category) {
        this.blockingReasons = BlockingReasonVO.removeByCategory(blockingReasons, category);
        clearDimensionByCategory(category);
        recompute(category.toTriggerEvent());
    }

    public void recompute(AvailabilityTriggerEventVO triggeringEvent) {
        boolean before = this.available;
        this.available = blockingReasons.isEmpty()
                && !lifecycleBlocked
                && !inspectionOverdue
                && !maintenanceOverdue
                && !calibrationBlocked
                && !malfunctionActive
                && !lifeExceeded
                && !commLost
                && !criticalAlarm;
        if (before != this.available) {
            registerEvent(new AssetAvailabilityChangedEvent(assetId, this.available, blockingReasons, triggeringEvent));
        }
    }
}
```

| 领域行为 | Java 方法 | 校验 INV | 发出事件 | 备注 |
|----------|-----------|----------|----------|------|
| `recompute(triggering_event)` | `AvailabilityProjectionModel.recompute(...)` | INV-13, INV-14, INV-27 | `AssetAvailabilityChangedEvent`（仅变化时） | 幂等发布 |
| `markUnavailable(reason)` | `AvailabilityProjectionModel.markUnavailable(...)` | INV-16, INV-17, INV-25 | 可能 `AssetAvailabilityChangedEvent` | 硬阻塞/条件阻塞都进入投影 |
| `clearBlocking(reason_type)` | `AvailabilityProjectionModel.clearBlocking(...)` | INV-13, INV-25 | 可能 `AssetAvailabilityChangedEvent` | 不直接恢复资产聚合 |
| `clearBlockingByCategory(prefix)` | `AvailabilityProjectionModel.clearBlockingByCategory(...)` | INV-15, INV-25 | 可能 `AssetAvailabilityChangedEvent` | 用于 `CALIBRATION_*` 批量清除 |

### 2.5 ScrapApplicationModel（报废申请）

**来源**：领域建模 §1.5
**一致性边界**：单次报废申请从提交到审批完结的流程与审批历史。
**实现形态**：`domain.model.scrap.ScrapApplicationModel`（充血聚合根）

```java
public class ScrapApplicationModel {
    private ApplicationIdVO applicationId;
    private AssetIdVO assetId;
    private AssetKindVO assetKind;
    private ScrapSourceVO source;
    private String sourceEventId;
    private String reason;
    private MoneyVO residualValue;
    private String applicant;
    private List<AttachmentVO> attachments;
    private ApprovalFlowVO approvalFlow;
    private ScrapApplicationStatusVO status;
    private DateTimeVO submittedAt;
    private DateTimeVO decidedAt;
    private String decidedBy;
    private Long version;
    private final List<Object> domainEvents = new ArrayList<>();

    public void approve(String approver) {
        requireStatus(ScrapApplicationStatusVO.SUBMITTED);
        ApprovalFlowVO advanced = approvalFlow.approveCurrentStep(approver);
        this.approvalFlow = advanced;
        if (advanced.isCompleted()) {
            this.status = ScrapApplicationStatusVO.APPROVED;
            this.decidedBy = approver;
            this.decidedAt = DateTimeVO.now();
            registerEvent(new ScrapApplicationApprovedEvent(applicationId, assetId, approver, advanced.allApprovers()));
        } else {
            registerEvent(new ScrapApprovalStepAdvancedEvent(applicationId, assetId, advanced.currentStep()));
        }
    }

    public void reject(String approver, String reason) {
        requireStatus(ScrapApplicationStatusVO.SUBMITTED);
        this.status = ScrapApplicationStatusVO.REJECTED;
        this.decidedBy = approver;
        this.decidedAt = DateTimeVO.now();
        registerEvent(new ScrapApplicationRejectedEvent(applicationId, assetId, approver, reason, approvalFlow.currentStep()));
    }

    public void withdraw(String applicant) {
        requireApplicant(applicant);
        requireStatus(ScrapApplicationStatusVO.SUBMITTED);
        this.status = ScrapApplicationStatusVO.WITHDRAWN;
        this.decidedBy = applicant;
        this.decidedAt = DateTimeVO.now();
        registerEvent(new ScrapApplicationWithdrawnEvent(applicationId, assetId, applicant));
    }
}
```

## 5. 应用服务与事务边界

应用服务命名以 `AppService` 结尾，事务放应用层；写仓储在领域服务内发生。每个写用例遵循：幂等 → 权限/审计 → 只读加载或构造 Info → 调领域服务 → 收集事件 → 同事务写 Outbox。

public class AssetRegistrationDomainService {
    public List<Object> registerAsset(RegisterAssetInfo info) {
        AssetIdentifierVO identifier = assetIdentifierClient.queryNextIdentifier(info.assetKind());
        assertIdentifierNotExists(identifier);

        List<Object> events;
        if (info.assetKind().isEquipment()) {
            EquipmentModel equipment = EquipmentFactory.create(info, identifier);
            equipmentRepository.insert(equipment);
            events = equipment.pullDomainEvents();
        } else if (info.assetKind().isFixture()) {
            FixtureModel fixture = FixtureFactory.create(info, identifier);
            fixtureRepository.insert(fixture);
            events = fixture.pullDomainEvents();
        } else {
            MeasuringInstrumentModel instrument = MeasuringInstrumentFactory.create(info, identifier);
            measuringInstrumentRepository.insert(instrument);
            events = instrument.pullDomainEvents();
        }

        AvailabilityProjectionModel projection = AvailabilityProjectionFactory.initial(info.assetId());
        availabilityProjectionRepository.insert(projection);
        events.addAll(projection.pullDomainEvents());
        return events;
    }
}

### 6.2 AvailabilityRecomputeDomainService

```java
public class AvailabilityRecomputeDomainService {
    public List<Object> onCalibrationFailed(DomainEventConsumeInfo info) {
        AssetHandle asset = assetRepositoryRouter.queryByAssetId(info.assetId());
        AvailabilityProjectionModel projection = availabilityProjectionRepository
                .queryByAssetId(info.assetId())
                .orElseThrow(AvailabilityProjectionNotFoundException::new);

        BlockingReasonVO reason = BlockingReasonVO.calibrationFailed(info.eventId(), info.occurredAt());
        if (asset.isInService()) {
            asset.suspend(reason);
        } else if (asset.isSuspended()) {
            asset.addBlockingReason(reason);
        } else {
            asset.rejectIfRetired();
        }
        projection.markUnavailable(reason);

        assetRepositoryRouter.update(asset);
        availabilityProjectionRepository.update(projection);
        return DomainEventCollector.merge(asset.pullDomainEvents(), projection.pullDomainEvents());
    }

    private List<Object> recomputeAndDecideReinstate(AssetHandle asset, AvailabilityProjectionModel projection) {
        projection.recompute(AvailabilityTriggerEventVO.external());
        if (projection.isAvailable() && asset.isSuspended()) {
            asset.removeAllClearedHardReasons(projection.blockingReasons());
            asset.reinstate();
        }
        assetRepositoryRouter.update(asset);
        availabilityProjectionRepository.update(projection);
        return DomainEventCollector.merge(asset.pullDomainEvents(), projection.pullDomainEvents());
    }
}
```

**关键实现点**：
- 已 `SUSPENDED` 资产叠加硬阻塞时必须调用 `addBlockingReason()`，不重复 `suspend()`，对应 INV-26。
- 所有可能恢复的事件都走 `recomputeAndDecideReinstate()`，不能由单个检定通过/维修完成事件直接 `reinstate()`。
- `AssetAvailabilityChangedEvent` 由 `AvailabilityProjectionModel` 比较前后状态后产生，无变化不发布。
- 计量证书过期除 MQ 事件外，还由定时扫描生成内部 `CalibrationExpired` 消费 Info，避免时间驱动遗漏。

### 6.3 ScrapApprovalDomainService

```java
public class ScrapApprovalDomainService {
    public List<Object> approveApplication(DecideScrapApplicationInfo info) {
        ScrapApplicationModel application = scrapApplicationRepository
                .queryById(info.applicationId())
                .orElseThrow(ScrapApplicationNotFoundException::new);
        application.approve(info.operator());
        List<Object> events = application.pullDomainEvents();

        if (DomainEventCollector.contains(events, ScrapApplicationApprovedEvent.class)) {
            AssetHandle asset = assetRepositoryRouter.queryByAssetId(application.assetId());
            asset.retire();
            assetRepositoryRouter.update(asset);
            events.addAll(asset.pullDomainEvents());
        }

        scrapApplicationRepository.update(application);
        return events;
    }
}
```

**关键实现点**：
- `submitScrapApplication` 先通过 `ScrapApplicationRepository.countPendingByAssetId` 查询，再依赖 `uk_pending_scrap` 兜底，保证 INV-04。
- 最终审批通过后，`ScrapApplicationModel.approve()` 与资产 `retire()` 在同一应用事务内提交，保证 INV-18。
- 驳回和撤回都会发布 `eam.asset.scrap` 事件，来源域通过 ACL 消费后恢复自身流程。

### 7.2 Outbox 设计

```text
聚合根 registerEvent
        ↓
DomainService 调用聚合行为 + 写 Repository，返回事件
        ↓
AppService 收集事件，同事务写 outbox_event
        ↓
OutboxPublisher 异步查询 INIT/FAILED 事件并发布 MQ
        ↓
Broker ack 后 update outbox_event.status=PUBLISHED
        ↓
失败按 retry_count + next_retry_at 重试，超过阈值进入人工补偿
```

| 字段 | 说明 |
|------|------|
| event_id | 全局唯一事件 ID，事件消费幂等键 |
| aggregate_id | asset_id / application_id / rule_id |
| aggregate_type | Equipment / Fixture / MeasuringInstrument / AvailabilityProjection / ScrapApplication / FixtureMatchingRule |
| event_type | Java 事件类名或稳定 eventType |
| topic | `eam.asset.lifecycle` / `eam.asset.availability` / `eam.asset.specification` / `eam.asset.metric` / `eam.asset.location` / `eam.asset.scrap` |
| payload | JSON 载荷，使用事件类逐字段序列化 |
| status | INIT / PUBLISHED / FAILED |
| retry_count | 发布重试次数 |
| next_retry_at | 下次重试时间 |
| occurred_at | 领域事件发生时间 |
| created_at / updated_at | Outbox 记录时间 |

**取舍说明**：不使用分布式事务；资产状态、报废审批、投影更新与 Outbox 写入在同一数据库事务内提交，MQ 发布由 OutboxPublisher 最终一致完成。

## 10. 异常、幂等与并发控制

| 场景 | 实现策略 | 对应 INV / 风险 |
|------|----------|-----------------|
| 重复资产登记命令 | `command_idempotency(command_id)`；资产编码唯一索引 | INV-01，防重复入账 |
| 资产编码并发冲突 | 编码生成 Client + `uk_asset_identifier(code)`；冲突重试生成新号或拒绝 | INV-01 |
| 资产终态后命令到达 | 聚合 `requireNotRetired()` 抛 `AssetRetiredException`；Repository version 防覆盖 | INV-03 |
| 状态非法流转 | 聚合 `requireState(...)` 抛 `AssetStateException` | INV-05~INV-08 |
| 同一资产重复未决报废 | 领域服务查询 + `uk_pending_scrap(asset_id, pending_flag)` | INV-04 |
| 报废审批并发批准/驳回 | `ScrapApplicationModel.version` 乐观锁；冲突提示重新加载 | INV-18, INV-19 |
| 资产状态与可用性投影并发更新 | 资产聚合和投影均带 version；同一事务更新；失败后由消息重试 | INV-13, INV-27 |
| 外部事件重复投递 | `consumed_event(source, event_id)` 唯一索引；重复直接 ack | INV-14, INV-27 |
| 外部事件乱序 | 阻塞原因按 reasonType 幂等追加/清除；恢复统一 `recomputeAndDecideReinstate`，不因单个通过事件直接恢复 | INV-25, INV-26 |
| 迟到的检定失败/过期事件 | 事件 Info 带 occurredAt；领域服务比较证书签发时间，过旧事件只记录消费不覆盖新证书 | INV-15 |
| 工装计数 total_count 重复或回退 | 保存 last_known_total；delta = total - last_known；delta <= 0 时幂等忽略或记录异常 | INV-10 |
| 运行时长负增量 | `RunHourDeltaVO` 构造校验 delta > 0；聚合拒绝负数 | INV-09 |
| 翻修清零与计数上报并发 | Fixture version 乐观锁；冲突后重放计数事件，确保 `accumulated_cycles >= reset_baseline` | INV-10, INV-12 |
| Outbox 发布失败 | `status=FAILED` + retry_count + next_retry_at；超过阈值进入补偿任务 | 事件最终一致 |
| MQ 发布成功但标记失败 | 下游按 event_id 幂等；Outbox 重发可接受 | 事件至少一次投递 |
| 证书时间驱动过期遗漏 | `CalibrationExpired` 事件 + 定时扫描 `cert_valid_until <= now()` 双触发，消费去重 | INV-15 |

**异常分层**：
- `AssetRegistryException`：utility 层全局业务异常基类，无具体业务判断。
- `AssetStateException` / `AssetIdentifierDuplicatedException` / `ScrapApplicationConflictException`：domain 抛出，facade-impl 转为统一错误码 DTO。
- `OptimisticLockConflictException`：repository 抛出，application 决定重试或返回并发冲突。

## 暂缓与扩展点

- 暂不引入 Event Sourcing：当前采用状态存储 + 领域事件 + Outbox + 操作流水即可满足 MES 台账追溯；未来如审计要求提升，可增加 `domain_event_log`，不改变聚合行为接口。
- 暂不拆独立 CQRS 读库：先用同库投影表和索引支撑资产列表、派工候选、规格查询；当可用性查询或追溯报表压力上升，再拆 Elasticsearch / ClickHouse。
- 字段级 Schema、单位归一化、资产编码规则细则暂由 `SpecificationProfileVO` 与 `AssetIdentifierClient` 承载扩展点，不在本实现设计内硬编码。
- 资产档案附件（说明书、图纸、合格证）后续通过 `AssetAttachmentClient` 接 OSS/文档服务；当前只在 `ScrapApplicationModel.attachments` 保留附件引用。
- ERP / PLM / 财务折旧同步只通过 ACL 适配，不进入领域聚合；财务月结、折旧规则不在本上下文实现。
- 点检、保养、检定、维修的计划/任务/SOP 不在本上下文实现；本上下文只消费结果事件并发布资产语义事件。
